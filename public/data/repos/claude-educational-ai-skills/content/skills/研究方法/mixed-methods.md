# 混合研究方法 (Mixed Methods Research)

## 技能描述
整合量化與質性研究方法，發揮兩種方法的優勢，對複雜的教育現象進行更全面深入的探究。

## 混合研究的價值

### 為什麼選擇混合研究？
```
量化研究優勢                 質性研究優勢
• 可推論母群                 • 深度理解
• 因果關係檢驗               • 脈絡掌握
• 客觀數據                   • 參與者觀點
• 可重複驗證                 • 彈性探索
        ↘                    ↙
           混合研究
         • 互補 (Complementarity)
         • 驗證 (Triangulation)
         • 拓展 (Expansion)
         • 解釋 (Explanation)
```

### 適用情境
- 量化結果需要質性解釋
- 質性發現需要量化驗證
- 研究問題具有多重面向
- 需要發展並測試新工具
- 政策研究需要數據與故事

## 混合研究設計類型

### 1. 並行設計 (Convergent Design)
```
        ┌─────────────────┐
        │     研究問題     │
        └────────┬────────┘
                 │
        ┌────────┴────────┐
        ↓                 ↓
  ┌───────────┐     ┌───────────┐
  │ 量化資料   │     │ 質性資料   │
  │  收集      │     │  收集      │
  └─────┬─────┘     └─────┬─────┘
        │                 │
  ┌─────┴─────┐     ┌─────┴─────┐
  │ 量化分析   │     │ 質性分析   │
  └─────┬─────┘     └─────┬─────┘
        │                 │
        └────────┬────────┘
                 ↓
        ┌─────────────────┐
        │   整合比較       │
        │   結果詮釋       │
        └─────────────────┘

時機：同時進行
目的：三角驗證、互補
權重：通常 QUAN = QUAL
```

### 2. 解釋性序列設計 (Explanatory Sequential)
```
        ┌─────────────────┐
        │     研究問題     │
        └────────┬────────┘
                 │
        ┌────────┴────────┐
        ↓                 │
  ┌───────────┐           │
  │ 第一階段   │           │
  │ 量化資料   │ ────────→ │ 量化結果
  │ 收集分析   │           │ 需要解釋
  └─────┬─────┘           │
        │                 │
        │ ←────────────────┘
        ↓
  ┌───────────┐
  │ 第二階段   │
  │ 質性資料   │ → 解釋量化結果
  │ 收集分析   │
  └─────┬─────┘
        ↓
  ┌───────────┐
  │  整合詮釋  │
  └───────────┘

時機：量化先、質性後
目的：解釋量化結果
權重：通常 QUAN > qual
```

### 3. 探索性序列設計 (Exploratory Sequential)
```
        ┌─────────────────┐
        │     研究問題     │
        └────────┬────────┘
                 │
        ┌────────┴────────┐
        ↓                 │
  ┌───────────┐           │
  │ 第一階段   │           │
  │ 質性資料   │ ────────→ │ 發展工具/
  │ 收集分析   │           │ 提出假設
  └─────┬─────┘           │
        │                 │
        │ ←────────────────┘
        ↓
  ┌───────────┐
  │ 第二階段   │
  │ 量化資料   │ → 驗證/推論
  │ 收集分析   │
  └─────┬─────┘
        ↓
  ┌───────────┐
  │  整合詮釋  │
  └───────────┘

時機：質性先、量化後
目的：探索建構→量化驗證
權重：通常 qual → QUAN
```

```

研究目的：
1. [目的1]
2. [目的2]

【選擇混合研究的理由】
我希望透過混合研究來：
- [ ] 用質性解釋量化結果
- [ ] 用量化驗證質性發現
- [ ] 全面理解複雜現象
- [ ] 發展並測試新工具
- [ ] 其他：[說明]

【研究條件】
研究對象：[描述]
可用時間：[期程]
研究資源：[列舉]

請設計：
1. 建議的混合設計類型與理由

5. 撰寫整合結果的建議
```

## 整合策略

### Joint Display（聯合展示）
```
┌──────────────────────────────────────────────────┐
│            研究發現聯合展示表                      │
├────────────┬────────────┬────────────────────────┤
│ 研究面向   │ 量化發現    │ 質性發現              │
├────────────┼────────────┼────────────────────────┤
│ 面向 A     │ 統計結果   │ 主題/引述             │
├────────────┼────────────┼────────────────────────┤
│ 面向 B     │ 統計結果   │ 主題/引述             │
├────────────┼────────────┼────────────────────────┤
│ 整合詮釋   │ 兩種結果如何互相支持/解釋/挑戰      │
└────────────┴────────────┴────────────────────────┘
```

### 結果對話類型
| 類型 | 說明 | 詮釋方向 |
|------|------|----------|
| 收斂 | 量化與質性結論一致 | 增強信心、三角驗證 |
| 互補 | 各自揭示不同面向 | 豐富理解、多元視角 |
| 歧異 | 結果不一致或矛盾 | 深入探究、修正理解 |
| 拓展 | 一方結果拓展另一方 | 新發現、後續研究 |

## 品質指標

### 混合研究品質檢核
```
量化品質
□ 效度（內容、建構、效標）
□ 信度（內部一致性、再測）
□ 統計結論效度
□ 外在效度（可推論性）

質性品質
□ 可信度 (Credibility)
□ 可轉換性 (Transferability)
□ 可靠性 (Dependability)
□ 可確認性 (Confirmability)

混合研究特有品質
□ 設計品質（適切性、嚴謹性）
□ 整合品質（整合程度、詮釋品質）
□ 整體推論品質
```

## AI/數位學習研究的混合設計範例

### 研究主題：AI 輔助差異化教學成效
```
探索性序列設計：

第一階段（質性）
- 訪談教師理解差異化教學困難
- 觀察課堂了解實施現況
- 發展 AI 輔助方案原型

第二階段（量化）
- 準實驗設計檢驗成效
- 前後測比較學習表現
- 問卷調查態度與滿意度

整合
- 將質性發現轉化為量化變項
- 用量化驗證質性觀察
- 整合說明 AI 輔助的作用機制
```
