# 實證研究自動化 (Empirical Research Automation)

## 技能描述

運用 AI 自動化實證研究的完整工作流程，從資料收集、程式碼轉換、統計分析到論文撰寫，大幅提升研究效率並確保研究可重現性。適用於教育科技、社會科學、心理學等需要量化分析的實證研究領域。

## 適用情境

- **複製經典研究**：將舊版統計軟體（Stata, SPSS）代碼轉換為現代 Python/R，延伸分析到最新資料
- **資料收集自動化**：自動爬取政府開放資料、學術資料庫、問卷平台等多元資料來源
- **統計分析加速**：快速生成描述性統計、推論統計與專業視覺化圖表
- **文獻回顧整理**：系統化搜尋、萃取、組織相關文獻並生成回顧章節草稿
- **論文撰寫輔助**：協助撰寫研究方法、結果、討論等章節，符合學術寫作規範
- **可重現性套件**：建立完整的 GitHub 專案，包含資料、程式碼、文檔與執行說明
- **準實驗資料分析**：處理教育介入研究的前後測資料，執行 ANCOVA 等統計方法

## 實證研究自動化工作流

```
階段 1：資料準備
  ├─ 程式碼現代化（Stata → Python/R）
  ├─ 網路資料爬取
  └─ 資料清理與整併
          ↓
階段 2：統計分析
  ├─ 描述性統計
  ├─ 推論統計（t檢定、ANOVA、迴歸）
  └─ 視覺化圖表（符合 APA 格式）
          ↓
階段 3：知識整合
  ├─ 文獻回顧自動化
  ├─ 理論框架建構
  └─ 研究缺口識別
          ↓
階段 4：論文產出
  ├─ 各章節草稿生成
  ├─ 學術語氣調整
  └─ GitHub 可重現性套件
```

### 📈 統計分析與視覺化

請為以下準實驗研究設計統計分析流程：

【研究設計】
[例如：非等組前後測設計（實驗組 vs 控制組）]

【變項資訊】
- 自變項（獨立變項）：[例如：教學方法（遊戲化 vs 傳統）]
- 依變項（結果變項）：[例如：學習成就後測、學習動機、批判思考]
- 共變項（控制變項）：[例如：學習成就前測、性別、年齡]

【樣本資訊】
- 實驗組樣本數：[例如：30 人]
- 控制組樣本數：[例如：28 人]
- 測量時間點：[例如：前測、立即後測、延宕後測（3 週後）]

【分析需求】
請選擇適當的統計方法並執行以下分析：

1. **描述性統計**
   - 各組在前測、後測的平均數、標準差
   - 檢查資料分布（直方圖、Q-Q 圖）

2. **推論統計**
   - 選擇適當統計方法（建議 ANCOVA，以前測為共變項）
   - 檢驗統計假設（常態性、變異數同質性、迴歸斜率同質性）
   - 計算效果量（partial η² 或 Cohen's d）

3. **視覺化圖表**
   - 組別比較長條圖（含誤差線）
   - 前後測變化折線圖
   - 符合 APA 第七版格式

請提供：
1. Python 或 R 完整分析腳本
2. APA 格式統計表格（可直接放入論文）
3. 高解析度圖表檔案（PNG 或 PDF）
4. 統計結果文字描述（適合論文結果章節）

【搜尋範圍】
- 時間範圍：[例如：2015-2024 年]
- 文獻類型：[例如：期刊論文、學位論文、研討會論文]
- 資料庫：[例如：Google Scholar, ERIC, 台灣期刊論文索引系統、華藝線上圖書館]

【文獻回顧焦點】
- ☐ 理論基礎（遊戲化理論、動機理論）
- ☐ 研究方法（準實驗設計、量表工具）
- ☐ 研究發現（成效、適用情境、限制）
- ☐ 研究缺口識別

請執行以下任務：

1. **搜尋策略規劃**
   - 建立布林搜尋字串
   - 設定納入/排除條件
   - 建議搜尋策略

2. **文獻萃取與組織**
   - 萃取每篇文獻的：研究目的、設計、樣本、主要發現、限制
   - 建立文獻矩陣表格（Excel 格式）
   - 依主題分類文獻（理論、方法、發現）

3. **撰寫文獻回顧**
   - 主題式組織（非逐篇摘要）
   - 批判性分析與綜合討論
   - 識別研究缺口與趨勢
   - 導出本研究的重要性

請提供：
1. 搜尋策略文件（布林字串與納入排除條件）
2. 文獻矩陣表格（Excel/CSV，包含所有萃取資訊）
3. 文獻回顧章節草稿（2000-3000 字）
4. 參考文獻清單（APA 第七版格式）

## AI 工具選擇建議

## 最佳實踐

### ✅ 研究倫理
- 確保資料收集符合 IRB（人體研究倫理委員會）規範
- 爬取資料時遵守網站使用條款，避免侵害著作權
- 保護個人隱私資料，進行去識別化處理
- 適當引用原始研究，避免學術剽竊

### ✅ 可重現性原則
- 使用版本控制（Git）追蹤所有程式碼與文件變更
- 記錄完整的計算環境（Python 版本、套件版本）
- 提供原始資料或資料獲取方法說明
- 撰寫清晰的程式碼註解與 README 文件

### ✅ 統計嚴謹性
- 報告完整的統計資訊（不僅是 p 值，包含信賴區間、效果量）
- 進行統計假設檢定（常態性、變異數同質性等）
- 執行敏感性分析與穩健性檢定
- 避免 p-hacking（反覆分析尋找顯著結果）

### ✅ AI 輔助寫作倫理
- 將 AI 視為寫作助手，所有內容需經人工審閱、修改、驗證
- 在論文致謝中說明使用 AI 工具的方式
- 確保引用正確且完整，避免 AI 生成的虛假引用
- 維持論文邏輯連貫性與學術誠信
