# 學習分析技術 (Learning Analytics)

## 技能描述
運用數據分析技術收集、分析與詮釋學生學習歷程數據，以優化教學決策、預測學習成效、提供個人化學習支援。

## 適用情境
- 線上學習平台數據分析
- 學生學習預警系統建置
- 教學成效評估
- 學習路徑優化
- 班級/學校層級的學習成效追蹤

## 學習分析層次

```
┌─────────────────────────────────────────────┐
│           描述性分析 (Descriptive)           │
│     「發生了什麼？」                         │
│     • 學習時間統計                           │
│     • 完成率追蹤                             │
│     • 成績分布                               │
├─────────────────────────────────────────────┤
│           診斷性分析 (Diagnostic)            │
│     「為什麼發生？」                         │
│     • 錯誤模式分析                           │
│     • 學習行為關聯                           │
│     • 影響因素探究                           │
├─────────────────────────────────────────────┤
│           預測性分析 (Predictive)            │
│     「可能會發生什麼？」                     │
│     • 學習成效預測                           │
│     • 退選/輟學風險                          │
│     • 學習困難預警                           │
├─────────────────────────────────────────────┤
│           處方性分析 (Prescriptive)          │
│     「應該怎麼做？」                         │
│     • 學習路徑建議                           │
│     • 介入策略推薦                           │
│     • 資源配置優化                           │
└─────────────────────────────────────────────┘
```

## 關鍵指標 (KPIs)

### 學習參與指標
| 指標 | 計算方式 | 參考值 |
|------|----------|--------|
| 登入頻率 | 每週登入次數 | 3-5次/週 |
| 學習時長 | 累計在線時間 | 依課程設計 |
| 完成率 | 已完成/總單元數 | >80% |
| 互動頻率 | 討論/提問次數 | 依班級規模 |

### 學習成效指標
| 指標 | 計算方式 | 警戒值 |
|------|----------|--------|
| 形成性評量平均 | 各次測驗平均 | <60分 |
| 進步幅度 | (後測-前測)/前測 | 負成長 |
| 概念掌握率 | 正確概念數/總概念數 | <70% |
| 作業繳交率 | 已繳交/應繳交 | <80% |

### 預警指標
| 風險等級 | 特徵 | 建議行動 |
|----------|------|----------|
| 高風險 | 連續3天未登入、成績低於30分 | 立即聯繫、個別輔導 |
| 中風險 | 完成率<50%、成績40-60分 | 關懷提醒、同儕支援 |
| 低風險 | 完成率<70%、成績60-70分 | 一般追蹤、鼓勵 |

## 提示詞範本

### 學習數據分析報告
```
請分析以下班級學習數據，生成分析報告：

【班級資訊】
班級：[班級名稱]
人數：[人數]
課程：[課程名稱]
分析期間：[起訖日期]

【數據摘要】
- 平均登入次數：[次數]
- 平均學習時長：[時數]
- 平均測驗成績：[分數]
- 作業繳交率：[百分比]
- 成績分布：[各分數段人數]

【問題描述】
[教師觀察到的現象或疑問]

請提供：
1. 數據趨勢分析
2. 學習困難學生辨識
3. 可能的原因診斷
4. 具體的教學調整建議
5. 下一步數據收集建議
```

### 個別學生學習診斷
```
請診斷這位學生的學習狀況：

【學生資料】
學生代碼：[代碼]
年級：[年級]

【學習行為數據】
- 登入時間分布：[描述]
- 平均每次學習時長：[分鐘]
- 影片觀看完成率：[百分比]
- 暫停/重播頻率：[頻率]

【學習成效數據】
- 各單元測驗成績：[列表]
- 錯誤題型分布：[類型與次數]
- 作業完成品質：[評語]

【比較基準】
- 班級平均：[相關數據]

請分析：
1. 學生的學習模式特徵
2. 優勢與待加強領域
3. 可能的學習障礙
4. 個人化學習建議
5. 教師/家長可採取的支援行動
```

### 學習預警模型設計
```
請協助設計一個學習預警系統：

【場域描述】
學校類型：[國小/國中/高中/大學]
學生規模：[人數]
課程類型：[必修/選修/線上課程]

【可用數據】
1. 學習平台數據：[列出可用欄位]
2. 成績系統數據：[列出可用欄位]
3. 其他數據：[如出缺席、輔導紀錄]

【預警目標】
希望預測：[如：期末成績不及格、中途退選]
預警提前時間：[如：期中考前2週]

請設計：
1. 預警指標選擇與權重
2. 風險等級定義
3. 預警觸發條件
4. 介入策略建議
5. 效果追蹤機制
```

## 偏鄉應用考量

### 數據收集挑戰
- 設備/網路限制導致數據不完整
- 學生共用設備造成識別困難
- 教師數位能力差異

### 因應策略
- 設計離線數據收集機制
- 簡化儀表板設計
- 提供教師培訓與支援
- 結合質性觀察紀錄

## 倫理與隱私

### 資料處理原則
- 最小必要原則：僅收集必要數據
- 匿名化處理：移除可識別資訊
- 知情同意：告知學生/家長
- 安全儲存：加密與存取控制

### 使用倫理
- 避免標籤化學生
- 數據輔助而非取代專業判斷
- 注意演算法偏見
- 確保公平使用
