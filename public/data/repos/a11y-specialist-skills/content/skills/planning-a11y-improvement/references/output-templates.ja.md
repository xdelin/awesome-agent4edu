# 出力テンプレート

このドキュメントは、planning-a11y-improvement スキルが生成する各成果物の詳細なテンプレートを提供します。

## 1. 成熟度評価レポート

### テンプレート

```markdown
## 成熟度評価

**現在のレベル**: L[1-4] [レベル名]

### 評価の根拠

| 軸 | 評価 | 根拠 |
|----|------|------|
| ガバナンス | L? | [具体的な根拠] |
| デザインシステム | L? | [具体的な根拠] |
| エンジニアリング | L? | [具体的な根拠] |
| QA/検証 | L? | [具体的な根拠] |
| トレーニング | L? | [具体的な根拠] |

### 強み
- [現時点での強み1]
- [現時点での強み2]

### ギャップサマリー
- [主要なギャップ1]
- [主要なギャップ2]
- [主要なギャップ3]

### 目標レベル
**12ヶ月後の目標**: L[目標レベル] [レベル名]
```

### 例

```markdown
## 成熟度評価

**現在のレベル**: L2 Repeatable

### 評価の根拠

| 軸 | 評価 | 根拠 |
|----|------|------|
| ガバナンス | L1 | 担当者は存在するが、公式なポリシーや予算なし |
| デザインシステム | L2 | 主要コンポーネントにa11y対応あり、ドキュメント不足 |
| エンジニアリング | L2 | PRレビューでa11yチェックあるが属人的 |
| QA/検証 | L1 | 手動チェックのみ、axe等の自動テストなし |
| トレーニング | L2 | 過去に勉強会実施、継続的な仕組みなし |

### 強み
- デザインシステムの基盤が存在する
- 一部のエンジニアにa11y知識がある
- 経営層がa11yの重要性を認識し始めている

### ギャップサマリー
- ライフサイクル全体でのQAゲートが未設置
- 計測・モニタリングの仕組みなし
- 組織横断的なオーナーシップが不明確

### 目標レベル
**12ヶ月後の目標**: L3 Managed
```

```markdown
### 遅行指標（Lagging Indicators）

| 指標 | 定義 | 目標値 | 測定方法 |
|------|------|--------|----------|
| [指標名] | [何を測るか] | [数値目標] | [測定手段] |
```

### 例

```markdown
## KPI/指標設計

### 先行指標（Leading Indicators）

| 指標 | 定義 | 目標値 | 測定方法 |
|------|------|--------|----------|
| a11y仕様定義率 | a11y要件が明記されたコンポーネント/全コンポーネント | 80% | デザインシステムドキュメントのカウント |
| レビューカバレッジ | a11yチェックを含むPR/全PR | 100% | CIログの集計 |
| トレーニング完了率 | 研修受講済み開発者/全開発者 | 100% | LMS記録 |
| 設計段階でのa11y考慮率 | a11y項目を含むデザインレビュー/全レビュー | 90% | デザインレビューログ |

### 遅行指標（Lagging Indicators）

| 指標 | 定義 | 目標値 | 測定方法 |
|------|------|--------|----------|
| 自動スキャン違反数 | axe-coreで検出されるLevel A/AA違反の総数 | 前月比-20% | CI/CDダッシュボード |
| 手戻り率 | a11y起因のバグ修正工数/全開発工数 | 5%以下 | Jiraチケット分析 |
| サポートチケット数 | a11y関連の問い合わせ件数 | 前月比-10% | サポートシステム集計 |
| 外部監査指摘件数 | 監査で発見された問題数 | 重大0件、中程度5件以下 | 監査レポート |
```

## 4. ステークホルダー説得資料

### テンプレート

```markdown
## ステークホルダー向け要点

### エグゼクティブサマリー
[1-2文で戦略の骨子]

### なぜ今取り組むべきか

#### リスク（対応しない場合）
- **法的リスク**: [具体的なリスク]
- **市場リスク**: [具体的なリスク]
- **レピュテーションリスク**: [具体的なリスク]

#### 機会（対応した場合）
- **市場拡大**: [具体的な機会]
- **競争優位**: [具体的な機会]
- **ユーザー価値**: [具体的な機会]

### 投資対効果

| 項目 | 今対応する場合 | 後で対応する場合 |
|------|----------------|------------------|
| [項目1] | [コスト/効果] | [コスト/効果] |

### 推奨アクション
1. [最初にすべきこと]
2. [次にすべきこと]
3. [その次にすべきこと]
```

### 例

```markdown
## ステークホルダー向け要点

### エグゼクティブサマリー
現在のアクセシビリティ成熟度はL2（Repeatable）で、体系的な取り組みへの移行が急務です。12ヶ月でL3（Managed）を目指し、法規制リスクの低減と市場拡大の両方を実現します。

### なぜ今取り組むべきか

#### リスク（対応しない場合）
- **法的リスク**: 2025年6月施行のEU European Accessibility Act (EAA) により、EU市場でのサービス提供に支障が出る可能性
- **市場リスク**: 公共機関や大企業の調達要件にa11y対応が含まれるケースが増加。未対応は入札資格喪失につながる
- **レピュテーションリスク**: 競合他社がa11y対応をマーケティングに活用し始めている

#### 機会（対応した場合）
- **市場拡大**: 世界人口の15%、約10億人が何らかの障害を持つ。高齢化により国内市場でも拡大中
- **競争優位**: 調達要件を満たすことで、公共機関や大企業への販路が開ける
- **ユーザー価値**: 対象市場の高齢化により、視覚・聴覚・運動機能が低下したユーザー層への対応が競合との差別化要因となる

### 投資対効果

| 項目 | 今対応する場合 | 後で対応する場合 |
|------|----------------|------------------|
| 修正コスト | 設計段階で対応：1x | リリース後に対応：10-30x |
| 市場機会 | EAA施行前に対応完了 | 施行後の緊急対応で販売機会損失 |
| 法的リスク | 計画的に低減 | 訴訟リスク増大、対応コスト上昇 |

### 推奨アクション
1. **今月中**: a11yポリシーの承認と担当者の正式任命
2. **3ヶ月以内**: 開発プロセスへのa11yチェック組み込み完了
3. **6ヶ月以内**: 主要機能のLevel AA準拠達成
```
