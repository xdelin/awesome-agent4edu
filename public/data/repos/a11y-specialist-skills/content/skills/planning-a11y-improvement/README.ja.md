# planning-a11y-improvement

[English](./README.md)

組織のアクセシビリティ改善計画を策定するスキル。成熟度評価、フェーズ別ロードマップ、KPI設計、ステークホルダー説得資料を生成します。

## アーキテクチャ

このスキルは**インタビュー駆動型**のワークフローを採用しています：

```
┌──────────────────────────────────────────┐
│   planning-a11y-improvement                  │
│   1. シナリオ特定                         │
│   2. 情報収集（インタビュー）             │
│   3. 成熟度評価                           │
│   4. 計画ドラフト生成                     │
│   5. レビューと調整（ユーザーと対話）     │
│   6. ファイル出力                         │
└──────────────────────────────────────────┘
                    │
                    ▼
          ┌─────────────────┐
          │   成果物         │
          │   - 成熟度評価   │
          │   - ロードマップ │
          │   - KPI設計      │
          │   - 説得資料     │
          │   → MDファイル   │
          └─────────────────┘
```

## 利用シナリオ

### 1. 新規導入フェーズ
アクセシビリティをこれから始める組織向け。ベースライン確立と基礎トレーニングを優先。

### 2. 改善加速フェーズ
既存の取り組みを体系化したい組織向け。ガバナンス強化とツールチェーン自動化を優先。

### 3. 外部監査対応フェーズ
法規制や監査への緊急対応が必要な組織向け。迅速なトリアージと法務連携を優先。

## 成果物

| 成果物 | 内容 |
|--------|------|
| 成熟度評価 | 現状のレベル（L1-L4）と根拠、ギャップ分析 |
| ロードマップ | フェーズ別の施策、担当、KPI |
| KPI/指標設計 | 先行指標・遅行指標の提案 |
| ステークホルダー説得資料 | ビジネスインパクト、リスク、投資対効果 |

## 参考ドキュメントの読み込み

これまでの取り組みをまとめたドキュメントを読み込んで、より精度の高い計画を策定できます。

**対応するドキュメント例：**
- アクセシビリティ試験結果
- これまでの取り組み履歴
- 既存のa11yガイドライン/ポリシー
- 技術スタックや組織体制の説明

情報収集フェーズでファイルパスを指定すると、内容を読み込んで成熟度評価やロードマップ作成に活用します。

## 使用方法

```
/planning-a11y-improvement
```

スキルが起動すると、以下の流れで計画を策定します：

1. **シナリオ特定**: 利用目的を確認
2. **情報収集**: 参考ドキュメントの読み込み + ビジネス、技術、組織の状況をヒアリング
3. **成熟度評価**: 5つの軸で現状を評価
4. **計画ドラフト生成**: ロードマップ、KPI、説得資料を生成
5. **レビューと調整**: ユーザーと対話して計画を洗練
6. **ファイル出力**: 最終版をMarkdownファイルとして保存
