# planning-wcag-audit

[English version](./README.md)

WAICのウェブアクセシビリティ試験実施ガイドラインに基づいて、WCAG準拠監査の計画を立てるスキル。

## アーキテクチャ

対話的な情報収集を行う**統合型パターン**:

```
┌─────────────────────────────────────┐
│   planning-wcag-audit               │
│   - サイト情報の収集                │
│   - 試験方法の選択                  │
│   - 対象ページの選定                │
│   - 監査計画書の生成                │
└─────────────────────────────────────┘
           │
    ┌──────┴────────┐
    │               │
    ▼               ▼
┌────────────┐  ┌─────────────┐
│ページ選定  │  │監査計画書   │
│ガイド      │  │テンプレート │
└────────────┘  └─────────────┘
```

## planning-a11y-improvementとの使い分け

| 観点 | planning-a11y-improvement | planning-wcag-audit |
|------|---------------------------|---------------------|
| **目的** | 組織のa11y改善ロードマップ | 監査実施の計画 |
| **スコープ** | 組織全体・長期的 | 特定サイト・短期的 |
| **出力** | 成熟度評価、KPI、戦略 | 対象ページリスト、試験環境、スケジュール |
| **参考資料** | 独自フレームワーク | WAIC試験実施ガイドライン |

## ワークフロー（5ステップ）

1. **サイト情報収集**
   - サイトの総ページ数（概算）
   - サイト構造（テンプレートの種類）
   - 対象適合レベル（A/AA）
   - 目的（準拠、一部準拠、配慮）

2. **試験方法選択**（WAICガイドラインに基づく）
   - **全ページ試験**: 100ページ以下のサイト
   - **ランダム選択**: 統計的サンプリング
   - **代表ページ選択**: 主要ページのみ
   - **組み合わせ方式**: 100ページ超のサイト（推奨）

3. **対象ページ選定**
   - ランダム選択のサンプル数:
     - 10ページ以下: 試行的試験
     - 11-24ページ: 最低限の判定基準
     - 25-39ページ: 標準的なレベル
     - 40ページ以上: 十分な判定基準
   - 代表ページの特定
   - ページタイプ別の分類

4. **試験環境確認**
   - ブラウザ・支援技術の組み合わせ
   - モバイル/デスクトップ
   - 試験ツール（オプション、代替可）

5. **監査計画書生成**
   - 試験概要（目的、範囲、方法）
   - 対象ページリスト
   - 試験環境
   - スケジュール目安
   - 担当者/役割
