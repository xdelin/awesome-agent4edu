# WCAG アクセシビリティチェックリスト

これは人間のレビュアー向けの参考資料です。Claudeは既にこれらのWCAG基準を知っています - これはドキュメント目的でのみ保守されています。

## 自動チェック項目

### セマンティック構造
- [ ] 見出し階層（h1-h6）が論理的で順序通り
- [ ] ランドマーク（main, nav, header, footer, aside）が存在
- [ ] HTML5セマンティック要素が適切に使用されている
- [ ] 文書言語が指定されている（`<html lang="...">`）

### 代替テキスト
- [ ] すべての`<img>`要素に適切な`alt`属性がある
- [ ] 装飾的な画像は`alt=""`または`role="presentation"`を使用
- [ ] アイコンボタンにアクセシブルなラベルがある（`aria-label`または`aria-labelledby`）
- [ ] SVGグラフィックに適切なタイトル/説明がある

### フォーム
- [ ] すべての`<input>`、`<select>`、`<textarea>`に関連付けられたlabelがある
- [ ] 必須フィールドが（色だけでなく）明示されている
- [ ] グループ化されたフォームコントロールにfieldset/legendが使用されている
- [ ] エラーメッセージがプログラム的に関連付けられている

### ARIA使用
- [ ] Roleが適切で冗長でない
- [ ] `aria-labelledby`と`aria-describedby`が存在するIDを参照している
- [ ] ARIAステート/プロパティが有効で正しく使用されている
- [ ] 矛盾したARIAがない（例: フォーカス可能な要素に`aria-hidden="true"`）

### キーボード・フォーカス
- [ ] すべてのインタラクティブ要素がキーボードでフォーカス可能
- [ ] フォーカス順序が視覚的な順序と一致
- [ ] 正の`tabindex`値（tabindex > 0）が使われていない
- [ ] フォーカスインジケーターが見える
- [ ] スキップリンクが存在する（必要な場合）

### リンク・ボタン
- [ ] リンクテキストが説明的（「こちら」だけではない）
- [ ] ボタンが`<button>`要素を使用（`<div onclick>`ではない）
- [ ] リンクとボタンに明確なアクセシブルネームがある
- [ ] 空のリンク/ボタンがフラグされている

### テーブル
- [ ] データテーブルがヘッダーに`<th>`を使用
- [ ] 複雑なテーブルが`scope`または`headers`/`id`の関連付けを使用
- [ ] `<caption>`または`aria-label`でテーブルの説明を提供

## 手動確認項目

### 色コントラスト
- テキストコントラストがWCAG AA（通常テキスト4.5:1、大きいテキスト3:1）を満たす
- UIコンポーネントコントラストが3:1を満たす
- リンクが色だけでなく区別可能

### キーボード操作性
- 完全な操作フローがキーボードのみで動作
- キーボードトラップがない
- すべての機能がキーボードで利用可能

### フォーカス管理
- モーダルダイアログが適切にフォーカスをトラップ
- クローズ時にトリガーにフォーカスが戻る
- SPAナビゲーションがフォーカスを管理

### 動的コンテンツ
- ARIAライブリージョンが更新をアナウンス
- 読み込み状態が伝達される
- エラーメッセージがアナウンスされる

## 重要度レベル基準

### Critical（致命的） - アクセスが完全にブロック
| パターン | 例 |
|----------|-----|
| 代替テキストの欠如 | 情報を持つ画像にaltがない、アイコンボタンにラベルがない |
| フォームラベルの欠如 | input/selectにラベルが関連付けられていない |
| キーボード操作不可 | クリックでしか動作しない要素、フォーカスできないインタラクティブ要素 |
| 致命的なARIAの誤用 | 存在しないIDへのaria-labelledby |
| コンテンツの非表示 | display:noneやaria-hidden="true"で重要な情報が隠されている |

### Major（重大） - アクセス可能だが大きな困難
| パターン | 例 |
|----------|-----|
| 見出し構造の欠如・破綻 | セクション見出しが見出し要素でない、レベル飛ばし |
| ランドマークの欠如 | main/nav/headerなどがない、ランドマークの重複 |
| 不適切なARIA | 間違ったrole、不正なaria-*属性値 |
| フォーカス順序の問題 | 視覚的な順序と大きく異なるtabindex |
| 不明確なリンク・ボタン | 「こちら」「詳細」のみ |
| テーブル構造の不備 | th/scopeがない |

### Minor（軽微） - アクセス可能で改善の余地
| パターン | 例 |
|----------|-----|
| より適切な要素選択 | divにonclickの代わりにbutton |
| 冗長なARIA | ネイティブセマンティクスと重複するrole |
| ベストプラクティス逸脱 | lang属性の欠如、スキップリンクがない |
| 軽微な見出し階層問題 | 軽微なレベルの不整合 |

## WCAG達成基準リファレンス

### レベル A
- **1.1.1 非テキストコンテンツ**: 画像の代替テキスト
- **1.3.1 情報及び関係性**: セマンティック構造
- **1.3.2 意味のある順序**: 読み上げ順序
- **2.1.1 キーボード**: キーボードアクセシビリティ
- **2.4.1 ブロックスキップ**: スキップリンク
- **2.4.2 ページタイトル**: 説明的なページタイトル
- **3.3.1 エラーの特定**: エラーメッセージ
- **3.3.2 ラベル又は説明**: フォームラベル
- **4.1.2 名前 (name)・役割 (role)・値 (value)**: アクセシブルネーム

### レベル AA
- **1.4.3 コントラスト（最低限）**: 色コントラスト 4.5:1
- **1.4.11 非テキストのコントラスト**: UIコンポーネントコントラスト 3:1
- **2.4.6 見出し及びラベル**: 明確な見出し/ラベル
- **2.4.7 フォーカスの可視化**: 視覚的なフォーカスインジケーター
- **3.2.3 一貫したナビゲーション**: ナビゲーションの一貫性

### レベル AAA
- **2.5.5 ターゲットのサイズ**: 44×44pxタッチターゲット
